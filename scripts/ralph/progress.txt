## Codebase Patterns
- Session validation: `getSessionUser(request)` from `src/lib/session.ts` — returns `SessionUser | null`
- Operators have `galpaoId` resolved from cargo; admins have `galpaoId = null` and must pass `?galpao_id=UUID`
- Supabase FK joins: `table(col1, col2)` in select — cast result with `as unknown as Type`
- Localizacao for separation comes from `siso_pedido_item_estoques` filtered by empresas in the operator's galpão
- Rate limiting for bip: `checkBipRateLimit(sessionId)` from `src/lib/session.ts`
- All API routes use `createServiceClient()` for DB, `logger` for logging, `NextResponse.json` for responses
- `siso_pedido_itens.quantidade_bipada` defaults to 0 (integer), `bipado_completo` defaults to false
- Operator-only endpoints guard with `if (!session.galpaoId)` → 403 (admins have null galpaoId)
- Pedido galpão ownership check: `pedido.separacao_galpao_id !== session.galpaoId` → 403
