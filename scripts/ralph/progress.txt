## Codebase Patterns
- Audio/UI utility files go in `src/components/separacao/` — no "use client" for pure TS utilities
- Use `sisoFetch` from `auth-context.tsx` for all API calls (adds X-Session-Id header)
- Tabs component in `src/components/ui/tabs.tsx` expects `Tab` type from `src/types/index.ts`
- All pages are `"use client"` except layout
- Tailwind classes directly, zinc-based palette, Lucide icons only
- Reuse `domain-helpers.ts` for ecommerce/decisao labels and colors
- `SeparacaoItem` (from item-separacao-row.tsx) and `PedidoSeparacao` (from pedido-separacao-card.tsx) are the canonical types for separacao UI
- GET /api/separacao returns snake_case fields (cliente_nome, not clienteNome) — match component props to API shape
- Bipar response includes `produto_id` — needed for desfazer-bip undo flow
- Use `inputMode="none"` on barcode scan inputs to prevent mobile keyboard from popping up
