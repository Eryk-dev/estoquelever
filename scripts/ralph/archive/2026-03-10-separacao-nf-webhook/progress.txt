## Codebase Patterns
- Tiny API v3 responses do NOT have a `{ data: ... }` wrapper — tinyFetch returns the raw shape
- `tinyFetch` already handles 429 retries, so new API functions just call it directly
- All Tiny API types go in `src/lib/tiny-api.ts` alongside their functions
- API JSON spec file is `api tiny.json` at project root (has spaces in name)
- The `origem` field on nota fiscal has `tipo` with values like "venda", "pedido_compra", etc.
- `siso_webhook_logs` dedup: set `dedup_key` on insert and catch error code 23505 (unique violation)
- For conditional DB updates, use `.update().eq(condition).select().maybeSingle()` to check if rows matched
- Webhook route.ts: empresa lookup is shared, then branches by `tipo` — NF webhooks skip codigoSituacao validation entirely
- For race-condition reconciliation between async events, query pending records + match via API call in a non-critical try/catch block

## 2026-03-10 - US-004
- Added NF webhook reconciliation logic to `src/lib/webhook-processor.ts` after pedido is saved
- Queries `siso_webhook_logs` for entries with `status = 'aguardando_pedido'` and `dedup_key LIKE 'nf_%'` for the same empresa
- For each pending NF webhook, calls `obterNotaFiscal` via Tiny API to match `origem.id` to the just-saved pedido
- If match found: transitions `status_separacao` from `aguardando_nf` → `pendente`, saves `url_danfe` and `chave_acesso_nf`, marks webhook as `processado`
- Wrapped in try/catch — reconciliation failure is non-critical and only logs a warning
- Files changed: `src/lib/webhook-processor.ts`
- **Learnings for future iterations:**
  - Reconciliation logic is defensive — in most cases there won't be pending NF webhooks when processWebhook runs
  - The match requires a Tiny API call (`obterNotaFiscal`) because the NF webhook log only stores the NF ID, not the pedido ID
  - `origemToken` from the processWebhook scope is reused for the API call — no need for a new token
  - The conditional update `WHERE status_separacao = 'aguardando_nf'` ensures idempotency — if the pedido hasn't reached aguardando_nf yet, the update simply matches 0 rows
