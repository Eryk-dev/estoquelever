## Codebase Patterns
- PostgreSQL partial indexes cannot use `now()` — it's STABLE, not IMMUTABLE. Use regular indexes instead.
- Migration naming: `YYYYMMDD_description.sql` in `supabase/migrations/`
- All tables prefixed with `siso_`
- Use `gen_random_uuid()` for uuid PKs, `timestamptz` with `DEFAULT now()` for timestamps
- Apply migrations via Supabase MCP `apply_migration` tool (project_id: wrbrbhuhsaaupqsimkqz)
- siso_usuarios.id is uuid type
- Supabase FK joins in select: `table(col1, col2)` — returns object for single FK, array for multi
- For joined results, cast with `as unknown as Type` since Supabase client lacks join type inference
- To get auto-generated UUID back from insert: `.insert({...}).select("id").single()`
- Non-critical side-effects (session creation, logging) should never block the primary response
- In-memory rate limiting uses module-level Map — persists across requests in same Node.js process
- `sisoFetch()` from `auth-context.tsx` wraps fetch with X-Session-Id header for authenticated requests
