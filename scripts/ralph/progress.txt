## Codebase Patterns
- Migrations use `supabase/migrations/YYYYMMDD_description.sql` naming convention
- Apply migrations via Supabase MCP `apply_migration` tool (project_id: wrbrbhuhsaaupqsimkqz)
- `siso_pedidos.id` is `text` (not uuid), `siso_pedido_itens.quantidade_pedida` is `numeric` (not integer)
- `siso_pedidos.data` is `date` type (used for ordering in separation queries)
- Existing tables have deprecated `filial` columns alongside new `empresa_id` FKs
- Always use `IF NOT EXISTS` for index creation in shared DB environments
- `ProcessedItem` fields are spread into `siso_pedido_itens` upsert — adding a field auto-persists it
- `getProdutoDetalhe` is called twice: main loop (kit detection) + `enrichItemMultiEmpresa` (image fallback) — both paths should capture new fields
