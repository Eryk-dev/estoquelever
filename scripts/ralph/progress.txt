## Codebase Patterns
- Always use `IF NOT EXISTS` for ADD COLUMN and CREATE INDEX in migrations
- Migration files follow naming: `YYYYMMDD_description.sql` in `supabase/migrations/`
- Supabase project ID: `wrbrbhuhsaaupqsimkqz`
- CHECK constraints on siso_pedidos use named constraints (e.g., `siso_pedidos_status_separacao_check`)
- Old status_separacao constraint was last updated in `20260311_modulo_compras.sql` to include `aguardando_compra`
- Backfill migrations should handle NULL values explicitly (e.g., `status=concluido AND status_separacao IS NULL`)
- Cancellation logic is in `src/app/api/webhook/tiny/route.ts`, NOT in `webhook-processor.ts`
- For dual-write: search `.update()` and `.upsert()` on `siso_pedidos` for `status` or `status_separacao` fields

- `bipar-embalagem` uses PL/pgSQL RPC `siso_processar_bip_embalagem` â€” dual-write for that path requires a DB function update, not TypeScript
- `expedir/route.ts` sets `expedido` which is NOT in StatusUnificado CHECK constraint â€” needs separate handling when expedido is added to the unified model
